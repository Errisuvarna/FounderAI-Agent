# FounderAI — Multi-Agent AI Startup Builder

Turn a raw startup idea into a complete, investor-ready business blueprint using five
collaborating AI agents (CrewAI + Gemini 2.5 Flash), grounded with real tool calling and RAG.

---

## 200-Word Project Summary

FounderAI addresses a real bottleneck for first-time entrepreneurs: turning a raw idea into a
structured, credible business plan normally requires expertise across market research, finance,
technology, and growth strategy — skills few solo founders have all at once. FounderAI solves
this with five specialized CrewAI agents that collaborate sequentially, each building on the
last agent's output: a Market Research Analyst sizes the opportunity and scans real competitors
via live web search; a Financial Analyst runs actual unit-economics math through a deterministic
calculator tool; a Solutions Architect proposes a pragmatic MVP stack; a Growth Strategist defines
go-to-market and risk mitigation; and a Pitch Writer synthesizes everything into an executive
summary and slide-by-slide pitch outline. Every agent can also query a ChromaDB-backed RAG layer
seeded with curated startup frameworks (TAM/SAM/SOM methodology, SaaS unit-economics benchmarks,
GTM playbooks), grounding output in real information rather than pure LLM guesswork. The full
stack is production-shaped: FastAPI + MongoDB Atlas backend with JWT authentication, a React +
Vite + Tailwind frontend with a live agent-progress tracker, and Docker-based deployment configs
for Render (backend) and Vercel (frontend).

## API Endpoints

| Method | Endpoint | Description | Auth |
|---|---|---|---|
| POST | `/register` | Create a new user account, returns JWT tokens | No |
| POST | `/login` | Authenticate, returns JWT tokens | No |
| GET | `/profile` | Return the current authenticated user's profile | Bearer JWT |
| POST | `/generate` | Run the 5-agent CrewAI pipeline, returns the full blueprint | Bearer JWT |
| GET | `/health` | Liveness check | No |

Interactive API docs are auto-generated by FastAPI at `/docs` once the backend is running.

## Environment Variables

### Backend (`backend/.env`)

| Variable | Description |
|---|---|
| `MONGODB_URI` | MongoDB Atlas connection string |
| `MONGODB_DB_NAME` | Database name (default: `founderai`) |
| `JWT_SECRET_KEY` | Long random secret used to sign JWTs |
| `JWT_ALGORITHM` | Signing algorithm (default: `HS256`) |
| `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` | Access token lifetime (default: `60`) |
| `JWT_REFRESH_TOKEN_EXPIRE_DAYS` | Refresh token lifetime (default: `7`) |
| `GEMINI_API_KEY` | Google AI Studio API key for Gemini |
| `GEMINI_MODEL` | Model id passed to CrewAI's LLM (default: `gemini/gemini-2.5-flash`) |
| `GEMINI_EMBEDDING_MODEL` | Embedding model for RAG (default: `models/text-embedding-004`) |
| `SERPER_API_KEY` | Serper.dev key powering the real web-search tool (optional but recommended) |
| `CHROMA_PERSIST_DIR` | Local path for ChromaDB's persistent store |
| `FRONTEND_ORIGIN` | Allowed CORS origin (your deployed frontend URL) |
| `APP_ENV` | `development` or `production` |
| `APP_PORT` | Port uvicorn binds to locally (default: `8000`) |

### Frontend (`frontend/.env`)

| Variable | Description |
|---|---|
| `VITE_API_BASE_URL` | Base URL of the FastAPI backend (e.g. `http://localhost:8000` or your Render URL) |

## Run Locally — Exact Commands

### 1. Backend

```bash
cd backend
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env              # then fill in MONGODB_URI, JWT_SECRET_KEY, GEMINI_API_KEY, SERPER_API_KEY
uvicorn app.main:app --reload --port 8000
```

Backend now runs at `http://localhost:8000` (docs at `http://localhost:8000/docs`).

### 2. Frontend

```bash
cd frontend
npm install
cp .env.example .env              # set VITE_API_BASE_URL=http://localhost:8000
npm run dev
```

Frontend now runs at `http://localhost:5173`.

### 3. (Optional) Run both with Docker Compose

```bash
cp backend/.env.example backend/.env      # fill in real values
cp frontend/.env.example frontend/.env
docker compose up --build
```

## Deployment

### MongoDB Atlas

1. Create a free-tier cluster at [mongodb.com/atlas](https://www.mongodb.com/atlas).
2. Under **Database Access**, create a user with read/write permissions.
3. Under **Network Access**, add `0.0.0.0/0` (hackathon demo) or your Render's static IP for production.
4. Copy the connection string from **Connect → Drivers** and set it as `MONGODB_URI`.

### Backend → Render

1. Push this repo to GitHub.
2. In Render, create a **New Web Service**, connect the repo, and set the root directory to `backend`.
3. Render will detect the `Dockerfile` automatically (or use `render.yaml` via Blueprint deploy).
4. Add all backend environment variables listed above in the Render dashboard (Environment tab).
5. Set the health check path to `/health`.
6. Deploy — your API will be live at `https://<your-service>.onrender.com`.

### Frontend → Vercel

1. In Vercel, **Add New Project** → import the same GitHub repo.
2. Set the root directory to `frontend`.
3. Framework preset: **Vite**. Build command: `npm run build`. Output directory: `dist`.
4. Add environment variable `VITE_API_BASE_URL` = your Render backend URL.
5. Deploy — your app will be live at `https://<your-project>.vercel.app`.
6. Back in Render, update `FRONTEND_ORIGIN` to this Vercel URL and redeploy the backend so CORS allows it.

## Notes on Production Hardening (Beyond Hackathon Scope)

- `/generate` currently runs the crew synchronously inside a threadpool; for higher concurrency,
  move execution to a background task queue (Celery/RQ) with a job-status endpoint.
- Add rate limiting on `/generate` given LLM/API cost per call.
- Rotate `JWT_SECRET_KEY` via a secrets manager rather than plain environment variables in production.
- Add per-user startup history persistence and list/detail endpoints (schema for this already
  exists in `startup_repo.py` and is ready to expose via new routes).
